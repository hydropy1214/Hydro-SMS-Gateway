# HYDROPY SMS Gateway Platform — Full Architecture & Development Instructions

You are the lead software architect and senior full-stack engineer for this project.

Your responsibility is to build a complete, production-quality SMS Gateway platform called **HYDROPY**.

Do not create random features or files. First understand the complete system architecture, review the existing repository, and then implement features according to this specification.

The goal is to create a platform where Android phones act as SMS gateway devices controlled by a web portal.

---

# PRODUCT OVERVIEW

HYDROPY is a distributed SMS gateway management platform.

The system consists of:

1. Web Dashboard
2. Backend API Server
3. PostgreSQL Database
4. Android Gateway Application
5. Real-time communication layer
6. Campaign and message processing system

The Android phone is the SMS sending device.

The backend controls, schedules, tracks, and manages devices.

Architecture:

```
                 HYDROPY DASHBOARD
                         |
                         |
                   API SERVER
                         |
          --------------------------------
          |                              |
     PostgreSQL                  WebSocket Server
          |                              |
          --------------------------------
                         |
                 Android Gateway App
                         |
                    Android SMS API
                         |
                      SIM CARD
                         |
                    Mobile Network
                         |
                    Recipient
```

# DEVICE CONNECTION SYSTEM

Implement QR-based pairing.

Flow:

1. Admin clicks "Add Device".
2. Server creates:

   * Device ID
   * Secure token
   * QR code.
3. Android app scans QR.
4. App authenticates.
5. Device connects through WebSocket.
6. Device becomes ONLINE.

QR contains:

```
server URL
device ID
authentication token
```

---

# ANDROID GATEWAY APP

The Android app is a worker.

Responsibilities:

## Connection

* Scan QR
* Authenticate
* Maintain WebSocket connection
* Reconnect automatically

## Heartbeat

Every 30 seconds send:

```
battery
charging status
signal strength
network
SIM information

device status
```

## SMS Processing

Receive jobs:

```
reason:
```

# CAMPAIGN SYSTEM

User workflow:

1. Create campaign.
2. Upload CSV contacts.
3. Validate numbers.
4. Remove duplicates.
5. Generate message jobs.
6. Add jobs to queue.
7. Scheduler assigns jobs to devices.
8. Devices send SMS.
9. Track results.
10. Generate report.

# MESSAGE QUEUE SYSTEM

```

Result
```

# FINAL GOAL

The final HYDROPY platform should work like this:

1. User opens dashboard.
2. User downloads Android gateway app.
3. User connects phone using QR code.
4. Phone appears online.
5. User creates SMS campaign.
6. System creates message jobs.
7. Scheduler distributes jobs.
8. Android devices send SMS using SIM cards.
9. Devices report results.
10. Dashboard shows live progress and reports.

Build this as a clean, scalable, maintainable platform.
